## 二、待办清单

### 🔴 P0 — 审批硬缺口（让它从「能演示」到「能用」）

- [ ] **退回 / 驳回到指定节点（含退回发起人重提）**
  - 目标：驳回时可选「退回上一步 / 退回发起人」，被退回人修改表单后流程**继续往下**，而非直接结束。
  - 验收：A 发起 → B 退回发起人 → A 改表单重新提交 → 回到 B；轨迹完整可见。
  - 改动：`FlowTaskService` 增加 `rejectToActivity(taskId, targetActivityId, comment)`，用 `runtimeService.createChangeActivityStateBuilder().moveActivityIdTo(...)`；前端审批弹窗增加「退回到」选择；记录 `outcome=rejectBack`。

- [ ] **会签 / 或签（多实例）**
  - 目标：一个用户任务可配置多人会签（全部通过 / 比例通过）或或签（一人通过即可），串行 / 并行。
  - 验收：3 人会签，2 人同意 1 人驳回按配置得到正确结果。
  - 改动：设计器节点属性面板增加「多实例」配置（`collection` / `completionCondition` / 串并行）；写入 BPMN `multiInstanceLoopCharacteristics`；审批服务对多实例任务的 `outcome` 聚合。

- [ ] **加签 / 减签 / 委派**
  - 目标：运行时动态增加或移除审批人；委派（处理后退回委派人）。
  - 验收：审批人 A 把任务委派给 C，C 处理后任务回到 A 完成。
  - 改动：`taskService.delegateTask / resolveTask`；前置 / 后置加签用多实例或新建任务；前端按钮 + 记录。

- [ ] **审批时可编辑表单**
  - 目标：审批节点的表单字段可由审批人补填 / 修改（当前为只读展示）。
  - 验收：审批人填写「审批备注金额」等字段并写回流程变量。
  - 改动：`approve-modal.vue` 的 `FormRender` 去掉 `disabled`，提交时把 `getValidatedValues()` 并入 `variables`（后端已支持 `variables`）。

### 🟠 P1 — 易用性 / 配置

- [x] **组织架构选人组件**（2026-06-30 完成，支持人员 / 角色 / 部门，可混合）

## 三、已知技术约束 / 易踩坑

- **Flowable 版本**：必须 8.0.0+，7.x 不兼容 Spring Boot 4。ACT_* 表由 `flowable.database-schema-update` 自动建。
- **MapperScan**：`@MapperScan("cc.oofo.**.mapper")` —— Mapper 必须放在 `*.mapper` 包，否则启动期 bean 注入失败。
- **`Rs.ok(String)` 陷阱**：该重载是「仅 message」，返回裸 String 会落进 `message` 而非 `data`；String 载荷需显式 `setData()`。
- **候选组 = 角色**：用户的 role_id 直接作为 Flowable 候选组 ID（见 `FlowIdentity`）。
- **审批意见 / 抄送**：存自建表 `flow_task_record` / `flow_cc_record`（Flowable 8 移除了 Comment API）。
- **网关路由**：依赖流程变量 `outcome`（`approve` / `reject`），条件如 `${outcome=='approve'}`，反向走默认连线。
